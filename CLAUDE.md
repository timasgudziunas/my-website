# My Website

A personal brand platform that converts traffic from social media, search, and professional networks into long-term audience ownership through email capture, while documenting my growth, ideas, and entrepreneurial journey in public.

## Primary Goal

The number one goal of this website is to collect emails from followers, friends, recruiters, investors, potential partners, and other people who want to follow my journey.

Every major page and section should support one of these outcomes:
- Capture an email
- Build trust
- Increase curiosity
- Explain who I am
- Show momentum
- Create future opportunity

The website should make someone think:

“I want to keep following this person. Something interesting is unfolding here.”

## Brand Positioning

This is not a portfolio, resume site, or website for one specific product.

This site positions me as:
- A future entrepreneur
- A lifelong learner
- An ambitious builder
- A transparent storyteller
- Someone documenting growth, ideas, experiments, and projects in public

The site should make visitors feel like they are early to an unfolding journey.

## Tech Stack

- Next.js 14 with App Router
- TypeScript
- Tailwind CSS
- Supabase
- Vercel
- Resend

## Project Structure

- `src/app/` - Pages and API routes
- `src/components/` - Reusable UI components
- `src/components/ui/` - Small generic components like Button, Card, Input
- `src/lib/` - Utility functions and service connections
- `src/types/` - TypeScript types
- `src/content/` - Static content, essays, notes, or copy
- `src/config/` - Site metadata, navigation, and constants

## Conventions

- Use Tailwind for all styling
- Components use PascalCase, like `HeroSection.tsx`
- One component per file
- Use TypeScript for everything
- Keep components clean and readable
- Favor simple, composable components
- Avoid unnecessary dependencies
- Prioritize performance, SEO, and email conversion

## SEO Requirements

The site should be structured so future essays, notes, and project pages can rank on Google.

Use:
- Semantic HTML
- Strong metadata
- Clean URLs
- Fast page loads
- Descriptive page titles
- Open Graph metadata
- Structured content sections
